# Демо: приём USDT → сметание (локальный девнет)

Полный цикл «деньги пришли на много адресов → собрали всё в один кошелёк» —
на **локальном девнете (Ganache)**, где ETH бесплатный и бесконечный, поэтому
**кран не нужен**. Все транзакции и газ — настоящие, просто на локальной цепочке.

## Что происходит
1. Поднимается devnet и деплоится `MockUSDT` (ERC-20, 6 знаков) —
   [contracts/MockUSDT.sol](contracts/MockUSDT.sol).
2. Создаётся 12 адресов приёма; на 8 из них начисляется тестовый USDT
   (имитация оплат клиентов).
3. Кнопки в панели:
   - **⛽ Пополнить газ** — кладёт немного ETH на каждый адрес с USDT (газ для перевода).
   - **🧹 Сметание всех адресов** — с каждого адреса шлёт USDT в казну (treasury),
     **по одной транзакции на адрес** (реальность account-модели), и считает газ.
   - **🔁 Пере-засеять USDT** — начислить USDT заново, чтобы прогнать ещё раз.

## Запуск
```bash
cd demo
npm install
npm start            # http://localhost:8001
```
Первый старт ~5–10 сек (деплой токена + засев). Ключи адресов генерируются
в процессе из тестовой мнемоники и **никуда не выгружаются**.

## Что показывают числа
После сметания панель показывает суммарный газ и средний газ на адрес (реальные
значения из чейна), и пересчёт в доллары так, как это стоило бы в **настоящем
mainnet** при текущем и при 30 gwei газе — это и есть цифра для решения L1 vs L2.
На тестовом прогоне: 8 адресов, ~255 000 газа всего (~31 900 на адрес).

## Почему это отдельно от SHKeeper
Настоящее сметание адресов SHKeeper в реальной сети требует тестового ETH из
крана (его нельзя получить автоматически) и приватных ключей адресов (они
зашифрованы внутри SHKeeper). Этот девнет-демо показывает тот же механизм
end-to-end, полностью управляемо и без кранов. Это соответствует «build custom»
пути из [../docs/architecture.md](../docs/architecture.md) (CREATE2-форвардеры +
батчинг — следующий шаг, если нужен продакшн).
